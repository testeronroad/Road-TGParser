# Инструкция по запуску RoadParcer

**RoadParcer** — программа от **Road Soft** для локального каталога публичных каналов и супергрупп Telegram. Эта инструкция для пользователей, которые скачали проект архивом (**Code → Download ZIP** на GitHub) или получили копию файлов другим способом.

---

## Что нужно

- Компьютер с **Windows**, **macOS** или **Linux**.
- Установленный **Python 3.10 или новее** ([python.org](https://www.python.org/downloads/)). При установке на Windows отметьте опцию **Add Python to PATH**.
- Аккаунт **Telegram** (номер телефона) для входа при первой настройке.
- Доступ в интернет.

---

## Шаг 1. Распаковка

Распакуйте архив в удобную папку, например:

`C:\Users\ВашеИмя\RoadParcer`

или

`~/RoadParcer`

Внутри должны быть файлы `requirements.txt`, `login_telegram.py`, папка `app` и другие файлы из репозитория.

---

## Шаг 2. Терминал в папке проекта

**Windows (PowerShell):** Shift + ПКМ по папке → «Открыть окно PowerShell здесь» или откройте PowerShell и выполните:

```powershell
cd C:\путь\к\RoadParcer
```

**macOS / Linux:**

```bash
cd ~/RoadParcer
```

---

## Шаг 3. Виртуальное окружение (рекомендуется)

Изолирует библиотеки проекта от системных.

**Windows:**

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Если выполнение скриптов запрещено политикой, один раз выполните:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

Либо используйте **cmd**:

```cmd
python -m venv .venv
.venv\Scripts\activate.bat
```

**macOS / Linux:**

```bash
python3 -m venv .venv
source .venv/bin/activate
```

В начале строки терминала появится префикс `(.venv)`.

---

## Шаг 4. Установка зависимостей

С активированным окружением:

```bash
pip install -r requirements.txt
```

---

## Шаг 5. Ключи Telegram API

1. Откройте [https://my.telegram.org/apps](https://my.telegram.org/apps).
2. Войдите по номеру телефона.
3. Создайте приложение, если ещё не создано.
4. Скопируйте **api_id** (число) и **api_hash** (строка).

В папке проекта скопируйте файл **`.env.example`** в **`.env`** (именно такое имя, с точкой в начале).

Откройте `.env` в блокноте и укажите:

```env
API_ID=ваше_число
API_HASH=ваша_строка
```

Сохраните файл. Файл `.env` никому не пересылайте и не выкладывайте в открытый доступ.

---

## Шаг 6. Первый вход в Telegram

Один раз создаётся файл сессии в папке `data` (папка появится автоматически):

```bash
python login_telegram.py
```

Введите номер телефона, код из Telegram и при необходимости пароль двухфакторной аутентификации. Должно появиться сообщение об успешном входе.

---

## Шаг 7. Запуск веб-приложения

```bash
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Оставьте окно терминала открытым. Откройте браузер и перейдите по адресу:

**http://127.0.0.1:8000**

Остановка сервера: в терминале нажмите **Ctrl+C**.

Если порт 8000 занят, укажите другой, например `--port 8080`, и откройте соответствующий адрес.

---

## Работа в интерфейсе

- **Спарсить** — поиск по ключевому слову в Telegram и добавление найденных каналов/супергрупп в локальную базу.
- **Поиск и фильтр** — просмотр и отбор записей в базе.
- **Категория** — можно изменить вручную в списке.
- **CSV / Excel** — экспорт с учётом текущих фильтров.
- **Удалить** — удаление строки из базы.

Данные хранятся в файле базы (по умолчанию `data/channels.db`). Резервную копию этого файла можно копировать вручную.

## Частые проблемы

| Проблема | Что сделать |
|----------|-------------|
| `python` не найден | Установите Python с python.org, включите PATH или используйте `py` вместо `python` на Windows. |
| Ошибка про API_ID / API_HASH | Проверьте `.env` в корне проекта, без лишних пробелов и кавычек. |
| Нет сессии Telegram | Снова выполните `python login_telegram.py`. |
| Сайт не открывается | Убедитесь, что uvicorn запущен и адрес совпадает с портом в команде. |
